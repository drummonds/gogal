# Examples — Structure and Standards

How gogal examples are organised, documented, and built. Reference for creating or refreshing example pages.

## File structure per example

```
examples/NN_name/
  go/
    main.go               # Server entry point (//go:build !js)
    main_wasm.go           # WASM entry point (//go:build js && wasm)
    gen_screenshot.go      # Generates screenshot SVGs (//go:build ignore)
    go.mod                 # Module with replace directive for local gogal
  templates/
    index.html            # WASM demo page (standalone)
    app.js                # WASM JavaScript glue
  index.md                 # Documentation source (markdown)

docs/NN_name/
  index.html              # Rendered from index.md — NEVER overwritten by build
  demo.html               # Copied from templates/index.html (WASM demo)
  app.js                  # Copied from templates/app.js
  main.wasm               # Compiled WASM binary
  wasm_exec.js            # Go WASM runtime
  NN_*.svg                # Screenshot SVGs generated by gen_screenshot.go
```

Key rules:
- `main.go` must have `//go:build !js` to exclude from WASM builds
- `main_wasm.go` must have `//go:build js && wasm`
- `gen_screenshot.go` lives in `go/` (not the example root) to avoid module ambiguity
- `docs:build-wasm` copies demo assets but **never overwrites `index.html`**

## Styling conventions

**Annotations** — blue callout boxes for explaining design decisions and API details:

```html

  <strong>Key concept</strong> — explanation text.
</div>
```

**Screenshots** — actual SVG output from the example, wrapped in a screenshot figure:

```html
<figure class="screenshot">
  <img src="NN_variant.svg" alt="Description" style="...">
  <figcaption>Caption</figcaption>
</figure>
```

Unlike lofigui (which captures HTML pages via url2svg), gogal screenshots are the actual SVG chart output — generated by a `gen_screenshot.go` file in each example directory.

**Buttons** — link to source and optionally WASM demo:

```html

</div>
```

## Screenshot generation

Each example has a `gen_screenshot.go` with `//go:build ignore` that produces SVG files directly from the gogal API. This keeps screenshots in sync with the library output.

```go
//go:build ignore

package main

import (
    "os"
    "codeberg.org/hum3/gogal"
)

func main() {
    chart := gogal.NewLineChart(/* ... */)
    chart.Add("series", points)
    f, _ := os.Create("../../../docs/NN_name/NN_variant.svg")
    chart.Render(f)
    f.Close()
}
```

Run all screenshots: `task docs:screenshots`

## WASM demo pattern

The WASM entry point exports a `goRender()` function that returns SVG HTML, then blocks forever. The JS glue calls `goRender()` once on `wasmReady` and inserts the result.

```go
//go:build js && wasm

func main() {
    js.Global().Set("goRender", js.FuncOf(func(this js.Value, args []js.Value) any {
        return renderHTML()
    }))
    js.Global().Call("wasmReady")
    select {}
}
```

For interactive examples (HTMX, live), the JS side can poll `goRender()` on an interval instead.

## Build process

### docs:build (markdown to HTML)

Add one line per example to the `docs:build` task in `Taskfile.yml`:

```yaml
- task-plus md2html --file examples/NN_name/index.md --dst docs/NN_name
```

### docs:build-wasm (compile + copy assets)

Add per-example block to the `docs:build-wasm` task:

```yaml
- mkdir -p docs/NN_name
- cd examples/NN_name/go && GOOS=js GOARCH=wasm go build -o ../../../docs/NN_name/main.wasm .
- cp "$(go env GOROOT)/lib/wasm/wasm_exec.js" docs/NN_name/
- cp examples/NN_name/templates/index.html docs/NN_name/demo.html
- cp examples/NN_name/templates/app.js docs/NN_name/
```

### docs:screenshots

Add each example's screenshot generator:

```yaml
- cd examples/NN_name/go && go run gen_screenshot.go
```

## Adding a new example

1. Create `examples/NN_name/go/` with `main.go` (`//go:build !js`), `main_wasm.go`, `gen_screenshot.go`, `go.mod`
2. Create `examples/NN_name/templates/` with `index.html` and `app.js`
3. Create `examples/NN_name/index.md` following the template above
4. Create `docs/NN_name/` directory
5. Add lines to Taskfile: `docs:build`, `docs:build-wasm`, `docs:screenshots`
6. Link the example number in `docs/index.md` table: `[NN](NN_name/)`
7. Run `task docs:screenshots && task docs:build-wasm && task docs:build`
8. Verify with `tp pages`
